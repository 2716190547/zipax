# PRD: 自动更新下载体验优化

## 背景

zipax 当前在检查到新版本后会触发前端运行时错误：

```text
Command plugin:dialog|confirm not allowed by ACL
```

根因是自动检查和手动检查流程使用了 `window.confirm`。在 Tauri 环境中，这类系统确认框会进入 dialog 插件权限链路；启动自动检查时一旦权限不匹配，用户会看到“zipax 启动失败”，更新提醒反而变成阻断启动的问题。

## 目标

- 检查到新版本后不再使用系统确认框。
- 在应用页面内展示轻量更新下载界面。
- 用户可以明确看到当前版本、新版本和下载动作。
- 下载完成后通过安装包完成覆盖安装，重新打开 zipax 即可更新。
- 自动检查失败保持静默，不影响启动和主流程。

## 非目标

- 本阶段不实现后台静默下载安装。
- 本阶段不引入 Tauri updater 插件的签名更新链路。
- 本阶段不自动退出、自动重启或自动替换正在运行的应用。

## 用户故事

- 作为用户，我希望 zipax 启动时即使检查更新失败，也不会影响正常打开应用。
- 作为用户，我希望发现新版后看到一个清楚的下载提示，而不是系统弹窗。
- 作为用户，我希望点“下载更新”后打开对应安装包或 release 页面，安装后重新打开即可完成更新。
- 作为用户，我希望暂时不更新时可以关闭提示，不影响继续使用。

## 交互方案

### 自动检查

1. 用户开启“自动检查更新”。
2. 应用启动约 3.5 秒后请求 `appcast.xml`。
3. 若当前版本已是最新，不展示任何提示。
4. 若发现新版本，在当前页面顶部展示应用内更新面板。
5. 若请求失败，静默记录到控制台，不打断启动。

### 手动检查

1. 用户在“通用 > 软件更新”点击检查按钮。
2. 检查中展示“正在检查”。
3. 若已是最新，展示当前版本已最新。
4. 若发现新版本，通用页状态展示新版本，同时打开应用内更新面板。
5. 若检查失败，在通用页展示错误信息。

### 更新面板

面板包含：

- 新版本号。
- 当前版本号。
- 更新说明文案：下载安装包后重新打开 zipax 即可完成更新。
- “下载更新”按钮：打开 appcast 中的下载链接。
- “稍后”按钮：关闭本次提示。

## 验收标准

- 启动自动检查发现新版本时，不再出现 `plugin:dialog|confirm not allowed by ACL`。
- 代码中自动更新流程不再调用 `window.confirm`。
- 自动检查失败不会阻断启动。
- 手动检查发现新版后，用户能看到应用内下载面板。
- 点击“下载更新”可以打开现有 appcast/release 下载链接。
- `npm run build` 通过。

## 后续可选增强

- 接入 Tauri updater 插件，实现签名校验、后台下载和应用内重启安装。
- 根据系统架构自动选择 `.dmg`、`.msi`、`.deb`、`.rpm` 或 `.AppImage`。
- 在下载界面展示 release notes 摘要。
- 增加“忽略此版本”。
