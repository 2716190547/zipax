# zipax 网站视觉与动效规范

这份规范用于约束 zipax 官网后续迭代，目标是保持“简洁、清爽、工具感、带一点品牌生命力”的统一体验。

## 1. 设计关键词

- 清爽：浅背景、大留白、弱边界、低噪点。
- 可信：标题粗、层级清楚、按钮和卡片反馈稳定。
- 品牌化：优先复用 zipax logo 的几何语言，包括斜切片、三角碎片、粒子聚合。
- 克制动效：动画用于解释层级和状态，不做炫技。

## 2. 页面层级

### 一级页面标题

- 用于下载、文档、支持等正式页面。
- 字重偏粗，建议 780-820。
- 行高控制在 1.08-1.15。
- 描述文字放在标题下方，不再把小标题放到大标题上面。

### 二级区块标题

- 用于首页功能区、流程区、CTA、文档正文模块。
- 字重 740-780。
- 描述文字使用 muted 色，保持 1.65-1.75 行高。

### 正文和辅助说明

- 正文默认 14-17px。
- 卡片说明最多 2-3 行，避免信息密度失控。
- 下载版本号、平台、文件大小属于辅助信息，使用 Chip 或 muted 小字。

## 3. 卡片规范

网站卡片统一分三类：

### ActionCard

用于下载包、支持入口等“可点击操作”。

- 圆角：`--card-radius`
- 图标容器：`--card-icon-radius`
- hover：上浮 2px，边框变 accent 混合色，出现轻微柔光层。
- focus-visible：必须有明显 outline。

### FeaturedCard

用于推荐下载、重点 CTA。

- 可以使用更明显的 accent-soft 背景。
- hover 上浮 3px。
- 内容结构保持：图标 / 文案 / 操作按钮。

### DocCard / 文档正文卡片

用于文档入口、步骤、排障、FAQ。

- 保持统一边框和圆角。
- 长内容优先分块，不把所有说明塞进一张大卡片。
- 排障模块可使用编号 Chip 表示步骤或问题顺序。

## 4. 动效规范

### 缓动曲线

推荐继续使用：

```css
--motion-ease: cubic-bezier(.16, 1, .3, 1);
--motion-ease-standard: cubic-bezier(.22, 1, .36, 1);
--motion-ease-exit: cubic-bezier(.4, 0, 1, 1);
```

### 时长建议

- hover：140-220ms
- 卡片进入：420-560ms
- 页面切换：500-700ms
- 首页首次过场：900-1200ms
- 背景呼吸：8-12s

### 进入顺序

页面进入建议遵循：

1. 页面容器出现
2. 标题出现
3. 描述出现
4. 重点卡片出现
5. 列表卡片错峰出现

错峰间隔建议 38-72ms。

## 5. 首页粒子规范

首页粒子 Logo 是品牌主视觉，不应承担过多信息。

- 默认状态下保持低频呼吸。
- 鼠标交互只做轻推，不要大幅炸开。
- 粒子数量按屏幕分级：
  - 桌面：约 680
  - 平板：约 440
  - 移动端：约 260
- 移动端优先保证稳定和居中，不追求复杂交互。

## 6. 页面过场规范

过场动画使用全屏遮罩，并结合 zipax logo 几何碎片。

- 首次进入首页：完整品牌过场。
- 普通路由跳转：短过场。
- 几何元素只做辅助，不能遮挡太久。
- 过场结束后页面内容应继续错峰进入。

## 7. 移动端规范

- 导航栏不覆盖全屏，使用原导航容器纵向展开。
- 展开后内容居中，但每个可点击项仍需保持足够点击区域。
- 卡片 padding 比桌面小一级。
- CTA 按钮移动端应占满宽度。

## 8. 深色模式与减少动画

### 深色模式

- 背景不要纯黑，保持深蓝灰。
- accent 可以更亮，但面积要克制。
- 阴影透明度提高，但避免重黑边。

### 减少动画

当用户开启 `prefers-reduced-motion: reduce`：

- 禁用长时间漂浮、粒子、几何碎片等装饰动画。
- 保留必要状态变化，但时长压缩到 1ms 或接近无动画。
- hover 不再产生位移。

## 9. 后续新增模块的检查清单

新增页面或模块前，检查：

- 是否使用现有 PageHeader / ActionCard / DocCard / FeaturedCard？
- 标题是否符合“标题在上，描述在下”的结构？
- hover/focus/reduced-motion 是否完整？
- 是否复用了 logo 几何语言，而不是临时画一套新图形？
- 移动端是否会过宽、过挤、或按钮不可点？
- 深色模式是否仍有足够对比？
