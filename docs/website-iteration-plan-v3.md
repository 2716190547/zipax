# zipax 网站迭代方案 v3

## 目标

将官网从“产品信息页”升级为简洁、清爽且有品牌记忆点的产品主页：首屏用 Logo 粒子聚合建立识别度，用轻量流体背景营造空间感，内容围绕用户任务而不是技术栈展开。

## 设计原则

1. **品牌先行**：Logo 粒子是唯一主视觉，其他动效降低存在感。
2. **内容克制**：每个区块只回答一个问题——是什么、能做什么、如何工作、为什么可信。
3. **动效有边界**：桌面端提供指针扰动；移动端降低粒子数量；系统开启“减少动态效果”时静态呈现。
4. **本地工具语气**：强调快速、本地、无需账号和不打断工作，不堆叠技术名词。

## 信息架构

| 区域 | 核心任务 | 主要内容 |
| --- | --- | --- |
| Hero | 建立第一印象并促成下载 | 产品定位、主下载、Logo 粒子聚合、三条信任点 |
| 能力卡片 | 快速理解价值 | 图片/PDF、文件夹自动化、本地处理、自动更新 |
| 工作流 | 降低上手门槛 | 选择 → 压缩 → 继续工作，搭配真实产品截图 |
| 桌面体验 | 建立信任 | 原文件安全、无需账号、免费开源 |
| 收尾 CTA | 再次促成下载 | 单一下载动作 |

## 分阶段路线

### Phase 1：品牌首页（已完成）

- Liquid Ether 风格的轻量 Canvas 氛围背景。
- 从真实 ZipaxIcon SVG 路径采样的 3D 粒子 Logo。
- 指针扰动、景深与缓慢自旋；移动端自动减少粒子。
- 首页内容重组为价值、工作流、桌面体验和 CTA。
- 浅色/深色主题、响应式与减少动态效果适配。

### Phase 2：内容可信度（建议下一迭代）

- 加入真实压缩前后案例，展示文件类型、原始体积、压缩后体积与耗时。
- 为 macOS、Windows、Linux 提供独立安装引导和系统要求。
- 将功能文案补齐至所有现有语言，避免非中英文页面回退英语。
- 增加隐私与本地处理说明页，并从首页信任点链接进入。

### Phase 3：转化与维护

- 增加下载按钮、平台和安装成功的匿名事件统计，不采集文件信息。
- 使用真实访问数据比较 Hero 文案与 CTA 表达，避免同时更改多个变量。
- 拆分 HeroUI 等较大依赖，降低首屏 JavaScript 体积。
- 建立 Lighthouse 基线和发布前视觉回归截图。

## 验收指标

- 首屏 5 秒内能明确回答 zipax 是什么、支持什么、为什么值得下载。
- 390px 宽度无横向溢出；核心按钮无需放大即可点击。
- `prefers-reduced-motion: reduce` 下背景和粒子保持静态。
- 浅色与深色主题下正文、边框和按钮均有清晰对比。
- 生产构建通过，浏览器控制台无错误或警告。

## 性能预算

- 粒子数量：桌面端不超过 920，移动端不超过 520。
- Canvas DPR：背景上限 1.5，粒子上限 2。
- 页面离开首页时卸载两套 Canvas 和动画循环。
- 下一阶段将主 JavaScript gzip 体积控制在 170 kB 以下。
